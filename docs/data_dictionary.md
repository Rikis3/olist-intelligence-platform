# Enterprise Data Dictionary

## Dimensions

### `dim_customers`
| Column | Type | Description |
|---|---|---|
| `customer_sk` | STRING | Primary Key. MD5 hash of `customer_unique_id`. |
| `customer_unique_id` | STRING | Business Key. Unique identifier for a human being (across multiple orders). |
| `zip_code_prefix` | STRING | First 5 digits of the customer's postal code. |
| `city` | STRING | Customer's city. |
| `state` | STRING | Customer's state code (e.g., SP for SĂ£o Paulo). |

### `dim_products`
| Column | Type | Description |
|---|---|---|
| `product_sk` | STRING | Primary Key. MD5 hash of `product_id`. |
| `category_name` | STRING | Translated English name of the product category (e.g., 'health_beauty'). |
| `weight_g` | INT64 | Physical weight in grams. |

## Facts

### `fact_orders`
| Column | Type | Description |
|---|---|---|
| `order_item_sk` | STRING | Primary Key. Hash of `order_id` + `order_item_id`. |
| `order_id` | STRING | Operational order ID. |
| `customer_sk` | STRING | FK linking to the customer who made the purchase. |
| `product_sk` | STRING | FK linking to the product sold. |
| `price` | FLOAT64 | The revenue generated by the item itself. |
| `freight_value` | FLOAT64 | The logistics cost paid by the customer for this item. |

### `fact_fulfillment`
| Column | Type | Description |
|---|---|---|
| `fulfillment_sk` | STRING | Primary Key. |
| `order_id` | STRING | Operational order ID. |
| `lead_time_days` | INT64 | Total days elapsed between purchase and delivery. |
| `is_sla_breach` | BOOLEAN | Flag indicating if actual delivery date exceeded the estimated delivery date. |

### `fact_reviews`
| Column | Type | Description |
|---|---|---|
| `review_score` | INT64 | Customer rating from 1 to 5. |
| `review_message` | STRING | Text feedback left by the customer. |
